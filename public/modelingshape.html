모델링 구조 설명

모델링 구조: 노란색 - 데이터 준비 단계, 파란색 - 모델 개발 단계, 녹색 - 모델 평가 단계, 빨간색 - 피드백 루프 및 모델 개선

과정

  • 데이터 준비 단계: 경기 데이터를 로드하고 필요한 특성들을 선택합니다. 날짜 정보를 연도와 월일로 분리하고, 팀 이름을 인코딩하며, 홈팀 결과를 승(0), 패(1), 무(2)로 인코딩합니다.
  • 특성 엔지니어링: 최근 5경기 평균 득점 계산, 공격 효율성 차이, 전반 골 수 차이 등의 새로운 특성 생성, 표준화를 통한 특성 스케일링 수행.
  • 오버샘플링: SMOTE-Tomek 기법을 사용하여 클래스 불균형 해결.
  • 특성 선택: XGBoost와 SHAP 값을 이용한 중요 특성 선택.
  • 모델 개발: XGBoost 분류기 사용 및 Optuna를 이용한 하이퍼파라미터 최적화.
  • 모델 평가: 정확도(Accuracy) 사용.

발견점

  • XGBoost 모델이 가장 좋은 성능을 보였습니다 (정확도 약 0.85).
  • 중요한 특성들: 홈팀과 원정팀의 유효슈팅 수, 공격 효율성 차이, 전반전 골 수 차이, 아디다스 포인트 (팀 평가 지표).
  • 신경망 모델(MLP)도 비슷한 성능을 보였지만, 약간 과적합 징후가 있습니다.

모델 강점

  • 체계적인 데이터 전처리와 특성 엔지니어링으로 모델 성능 향상.
  • SMOTE-Tomek을 통한 클래스 불균형 해결.
  • XGBoost와 Optuna를 활용한 강력한 모델 최적화.
  • 특성 중요도 분석을 통한 모델 해석 가능성 확보.

개선할 수 있는 부분

  • 교차 검증 단계 추가.
  • 다양한 모델 비교 (예: 랜덤 포레스트, 신경망 등과의 성능 비교).
  • 모델 앙상블 기법 적용 고려.
  • 시계열 특성을 더 활용할 수 있습니다 (예: 팀의 최근 성적 추세).
  • 앙상블 기법을 시도해볼 수 있습니다 (예: XGBoost와 신경망 모델의 결합).
  • 다양한 평가 지표 사용 (예: F1 점수, ROC AUC 등).
  • 특성 중요도에 기반한 더 세밀한 특성 선택.
  • 무승부 예측 정확도 향상을 위한 추가 전략 개발.
